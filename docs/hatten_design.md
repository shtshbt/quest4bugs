# 発展部門 設計

Quest4Bugs の「発展部門」は、受験レベルの難問を解かせる場ではなく、**各分野の基礎演習を手軽に大量反復できる**学習レイヤーとして設計する。入力は iPad のタップで完結させるため、答えは常に **1 つの整数**に収束する形式に限定する。難度を競うのではなく、基礎を繰り返し固めることを最優先とする。

## コンセプト

- 受験レベルの難問ではなく、各分野の**基礎演習を手軽に大量反復**することを主目的とする。
- 答えは**1 つの整数**で、iPad の**タップ入力**だけで解答できる問題に限定する。
- 難易度の高さより、**基礎演習の反復量**を重視する。
- **図形系は作図・描画が必要**なため、発展部門では**スキップ**する（タップ入力で整数回答に収まらないため）。

## コース別 棲み分け

発展部門は 2 コース構成とし、**カテゴリの重複が出ないように棲み分ける**。各カテゴリ内の難度差は、後述の**適応レベル（Lv1〜10）**が吸収するため、コース分けは「難度」ではなく「分野」で行う。

| コース | 基礎計算 | 発展演習 |
|---|---|---|
| k5（5 歳〜） | たし算筆算・ひき算筆算・九九・あんざん・**わり算（新）** | 和差算／文章整理・時間計算・掛け算文章題 |
| k10（受験） | 四則混合・工夫・小数・分数・まちがい | 濃度・旅人算系・比・つるかめ・過不足／差集め・平均・年齢・損益・仕事・植木・周期／日暦・流水・通過 |

棲み分けの方針:

- **和差算は 5 歳側（k5）に寄せる**。10 歳（k10）では同系統の問題を、**つるかめ算・消去算**などのより難しい型でカバーし、内容の重複を避ける。
- **5 歳の発展演習は当面 3 カテゴリ**（和差算／文章整理・時間計算・掛け算文章題）に絞る。
  - 将来的な拡張余地として、**簡単なつるかめ・植木・周期**を追加できる。

## レベル分類方針

全分野に共通して、**「サブタイプの追加＝難度上昇」**という考え方を適用する。1 つのカテゴリの中で、問題の型（サブタイプ）を段階的に切り替えることで Lv1〜10 の難度カーブを作る。

例: 旅人算

| Lv | サブタイプ |
|---|---|
| 1〜2 | 出会い |
| 3〜4 | 追いつき |
| 5〜6 | 往復・円周 |
| 7〜8 | 速度変更 |
| 9〜10 | 比・複合 |

- Lv 内のさらに細かい難度調整は、**カテゴリ内のパラメータ範囲**（数値の桁・係数の大きさ等）で行う。
- **適応レベル制**は全発展カテゴリに流用する。**10 問ごとに評価し、正答が多ければ +2 Lv（8 問以上＝8↑）、少なければ −1 Lv（6 問以下＝6↓）**の方針でレベルを自動調整する。

## ボス連動

- 発展カテゴリも**適応 Lv**を持つため、その到達度をボス解禁条件に組み込む。
  - 例: **「濃度 Lv7 到達」**を**受験ティアの後半ボス／モズの解禁条件**とする。
- **5 歳ティアは発展演習を解禁条件に含めない**。基礎計算のみで先に進めるようにし、年齢間の**公平性を維持**する。
