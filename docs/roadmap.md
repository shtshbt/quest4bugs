Quest4Bugs の今後の機能追加候補をまとめる。優先度・着手判断のためのバックログ。
着手時は各項目の「実装方針案」を起点に、既存資産（`_gen/` の生成スクリプト群・`shared/*.js`）を流用する。

## ① えいたんご：熟語（じゅくご）問題モードの追加

**目的**: 単語の暗記だけでなく、句動詞・連語・イディオム（look forward to / give up / be made of …）を
出題して実用的な英語力につなげる。各フィールド（にわ／いけ／はやし／やま／ジャングル／アマゾン）の
出題形式に「じゅくごモード」を追加する。

**現状・既存資産**:
- `shared/colloc.js` に「コロケーション/語法問題（英検形式・短文穴埋め）」データが既にある（`en -> {sentence, distractors[3]}`、全フィールド対象）。これは単語の語法確認用で、熟語そのものの出題ではない。
- えいたんご本体の出題・SRS は `eitango/index.html`（`SRS`=Leitner、`p.srs[key]=[box,due,seen,wrong]`）。

**実装方針案**:
1. 熟語データ層を新設（`shared/idiom.js` 等）。キー＝熟語、値＝`{ja訳, 例文(___入り), distractors, field/level}`。フィールド＝英検レベル対応に合わせて割り振る（既存の語彙レベル分担を踏襲）。
2. フィールド選択後のモード切替に「たんご／じゅくご」を追加。じゅくごモードは熟語データを出題プールにする。
3. SRS は単語と分離（熟語キーに `idiom|` などの接頭辞を付けて `p.srs` に同居、または別マップ）。バトル到達度（`reachEng`）への寄与方針も決める。
4. 新規熟語の必須セット：(1) 熟語データ本体、(2) ふりがな `shared/yomi.js`（ja訳の完全一致キーでルビ登録）、(3) 穴埋め用 distractor。

**留意点**: 「熟語」と既存「コロケーション」の役割を明確に分ける。バトル解禁への影響（じゅくご習得を到達度に含めるか）を `docs/battle_design.md` の到達度モデルと整合させる。

## ② えいたんご：準1級・1級 語彙リストのさらなる拡充

**目的**: 上位フィールド（ジャングル＝準1級・アマゾン＝1級）の語彙を実戦的な規模へ広げる。

**現状・既存資産**（2026-06 時点）:
- にわ256／いけ319／はやし256／やま242／どうくつ410／ジャングル(準1級)約1797／アマゾン(1級)約545。
- **重要な制約**: 1級語彙は LLM 生成だけだと約545で頭打ち。追加生成しても同じ難語に収束し純増が乏しい。
  1級を1000語規模にするには **外部の実在単語リスト（パス単・英検公式の出る順語彙 等）の reference が必須**。

**実装方針案**:
1. 外部の実在1級／準1級語彙リストを reference として用意（出典：英検公式「でる順パス単」相当の見出し語、CEFR C1相当リスト等）。**着手時にまず具体的なリスト案を提示**して、採用語を確定する。
2. 確定リストを元に、訳・例文・ルビ・コロケーション（穴埋め）を `_gen/` のworkflowスクリプト（`gen_colloc*.js` / `gen_ruby*.js` 等）で生成する。
3. 新規語の必須3点セット：(1) RAW配列 `[en,ja,ex,pos]`、(2) ふりがな `shared/yomi.js`（完全一致キー）、(3) コロケーション `shared/colloc.js`（無いと4択にフォールバック）。
4. 重複除去は既存プール（準1級は `P1A+P1B+P1C+P1D+PRE1RAW` 合算、1級は `ICHIRAW`）に対して行う。

**留意点**: 語の難度が実レベルに合っているか（収束した難語に偏らないか）を抽出元リストで担保する。フィールド表示語数のバランスも確認。

## ③ 漢字：「思い出し問題」モードの追加（えいたんごに倣う）

**目的**: えいたんごの「おもいだし問題」（一定期間あけて再出題する間隔反復／ごぶさた復習）を漢字にも導入し、
反復学習を促す。

**現状・既存資産**:
- えいたんご側は SRS（Leitner）と「おもいだし問題」モードが実装済み（`eitango/index.html`）。出題対象は `due`（再出題期日）に達した既習語。
- 漢字側（`kanji/index.html`）は学年別テスト・習得段階（🥚🐛🛡️🦋）はあるが、間隔反復ベースの「思い出し」モードは無い。

**実装方針案**:
1. 漢字の各問に再出題期日（`due` 相当）を持たせる。正解で間隔を延ばし、誤答で短縮（えいたんごの Leitner ロジックを流用）。
2. ホームに「おもいだし問題」導線を追加。`due` 到来分を優先出題する。
3. 既存の習得段階（🥚🐛🛡️🦋）と矛盾しないよう、思い出し問題は「習得済みの維持・再確認」に位置づける（新規習得カウントとは分離、易問周回での farming を防ぐ＝バトル到達度には寄与させすぎない）。

**留意点**: えいたんごと用語・演出（前進演出・「あと○かいで…」表記）を揃えて学習体験を統一する。漢字の保存データ構造（`kanji <profileId>`）に `due`/復習履歴を追加する際は後方互換に注意。

## ④ 計算：カテゴリ別 Lv 推移グラフ（未着手）

**目的**: 各計算カテゴリの Lv1→10 の上達過程を折れ線グラフで可視化し、達成感・継続動機につなげる。

**現状・前提**:
- `p.lv[cat]` は**現在のLvのみ**保持。過去の推移（日付→Lv）は未記録 → **Lv変化時のログ追記が前提**（実装後から記録開始、過去分は遡れない）。

**実装方針案**:
1. Lv変化時に履歴を追記（例 `p.lvlog[cat]=[{d:"YYYY-MM-DD",lv}]`、同日は最新で上書き）。後方互換に注意。
2. 記録画面にカテゴリ別の折れ線グラフUI（ポータルの日次バーと同方式）。
3. K10は32カテゴリと多いので、コース該当カテゴリのみ／タブ切替で表示。

**留意点**: 保存サイズ増に注意（日次1点/カテゴリに圧縮）。マスター虫(Lv10制覇)到達点をグラフ上にマーキングすると達成が映える。
